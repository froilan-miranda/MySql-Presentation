Database Basics

Database vs. Spreadsheets

Both databases and spreadsheets store data in rows and columns. Both can calculate new data based on existing data. And both can allow more than one person to view and change data.

Database vs. Spreadsheets

Spreadsheets

  • Data stored in a cell
  • Calculation or value
  • Contains formatting
  • Can move, sort data

Databases

  • Data stored in a record
  • Values stored
  • Formatting and calculations are applied later
  • Sorting is a display issue

Database vs. Spreadsheets

In a spreadsheet, everything you do results in data. If you do a calculation involving one cell, that calculation lives in another cell. In a database, you can do a calculation on a record and just show the calculation result without adding more data to the database. One of the biggest benefits of a database is that you can eliminate duplicate information.

Spreadsheets

  • Great for ledger type activities and lists
    bank transactions, calculating an account balance
  • Good for creating charts and graphs of data

Databases are More Efficient

  • Can give the data type
  • No redundancy for formatting differences
  • Less error prone
  • Other users do not change formatting/sorting

Relational Database vs. flat files

Similarities

  • Show and manipulate data
  • Stored data does not change
  • Extract data from fields
  • Matching and filtering

Similarities

  • No data types
  • Field seperation manually enforced
  • For writing
  • For layout
  • For moving text around

Normalizing databases

The objective is to isolate data so that additions, deletions and modifications of a field can be made in just one table, and then propagated through the rest of the database using the defined relationships.

Data types

  • VARCHAR - variable character (255 characters per field)
  • CHAR - character (fixed width)
  • TEXT - Variable-length (4 gigs per field)

Text Sizes - Bigger capacity = more overhead
https://dev.mysql.com/doc/refman/5.6/en/storage-requirements.html

Data types

  • DATE - YYYY-MM-DD, e.g. 1978-09-17
  • TIME - HH:MM:SS, e.g. 05:41:26
  • YEAR - YEAR(2) or YEAR(4)

A Moment in Time

  • TIMESTAMP
    • Limited range- 1970-01-01 through 2038-01-19
    • Can automatically populate
    • On row creation and/or update
  • DATETIME - Range is 1000-01-01 through 9999-12-31

Numbers with Decimals

  • DECIMAL(5,2)
    • 5 total digits
    • 2 digits after the decimal point
    • -999.99 to 999.99
    • Exact decimal storage
  • DATETIME - Range is 1000-01-01 through 9999-12-31
